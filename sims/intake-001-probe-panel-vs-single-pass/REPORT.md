# VERDICT 002 · INTAKE 001 — Panel-mode vs single-pass probing

## METHOD LABEL: `prototype` (measured prototype / spike — ladder rung 2)
No seedable dynamics: the question is whether adding a panel of ideation personas + a synthesizer changes an LLM probe's *recommendation* or *report quality* enough to justify multi-agent cost. That is only answerable by running both modes for real. So I built the smallest real head-to-head: **3 real backlog ideas × {single-pass ×3 reps, panel ×3 reps} = 18 live probe runs (54 sub-agents) + 6 blind pairwise judge runs**, every sub-agent instrumented for tokens (`subagent_tokens`) and wall-time (`duration_ms`) read from its `<usage>` block. Absolute token/wall numbers are Opus-4.8-in-this-environment specific and are reported as **indicative**; the **ratios** are the transferable measurement.

## What the prototype MEASURES
- **Mode 1 (single-pass):** one agent, one pass over a fixed 5-point probe battery → one JSON verdict {recommendation ∈ BUILD/HOLD/KILL, confidence, top_risk, cheapest_evidence, report}.
- **Mode 2 (panel):** N=3 ideation personas (skeptic / builder / user-value) probe independently and in parallel, then 1 synthesizer reconciles them into one verdict of the same schema. The docs specify "N personas + synthesizer" but pin no N and no roster; **N=3 and this roster are a documented choice of the reconstruction**.
- **3 real ideas** (idea-engine backlog): `discord-oauth-personal-stats`, `wild-encounters-activity-spawning`, `peer-karma-reputation-system`.
- **Per cell:** recommendation, confidence, agents (1 vs 4), tokens (summed over the 4 panel agents), wall-time (single agent; for the panel, max(3 parallel personas)+synth, modelling real parallel execution).
- **Quality:** 2 blind pairwise judges per idea, A/B order swapped, 5-criterion rubric that penalises fabricated specifics.
- **Honesty control:** 3 reps per mode per idea, to separate inter-mode difference from intra-mode LLM noise.

## What it SETTLED (the load-bearing claims — measured)
1. **Panel changes the recommendation on contested ideas, not on well-scoped ones.** The modal recommendation flipped on **2 of 3 ideas** — `peer-karma` (single-pass 3/3 BUILD → panel modal HOLD) and `discord-oauth` (single-pass 2:1 BUILD → panel 3/3 HOLD) — and did **not** flip on `wild-encounters` (both 3/3 BUILD), the one idea whose main axis a deterministic sim had already settled (`sims/intake-003-wild-encounter-spawn-tuning/`). **Both flips were BUILD→HOLD**; 0/2 went toward less caution. The panel's skeptic / user-value lenses systematically pull toward "measure before shipping."
2. **The karma flip exceeds intra-mode noise.** Single-pass was unanimous BUILD across all 3 reps (zero intra-mode variance) while the panel's modal verdict was HOLD — so the difference is the *mode*, not luck. On `discord-oauth` single-pass was itself split 2:1, so there the panel's role was **disambiguation** (it resolved the ambiguity to a stable unanimous HOLD) rather than a clean novel flip.
3. **Panel costs ~4× agents and ~3× tokens but only ~1.6× wall-time.** Measured means: single-pass ≈ **26.8k tokens / 1 agent / ~50.0s**; panel ≈ **81.5k tokens / 4 agents / ~80.5s**. Ratios: agents **4.00×**, tokens **3.05×**, wall **1.61×** — sublinear in wall because the 3 personas run in parallel.
4. **Blind judges slightly prefer the panel report, with no position bias.** Mode 2 won **6/6** pairwise comparisons, every margin "slight," and won in **both** A/B orderings for all 3 ideas — so the preference is not a slot artifact. The edge came from panel-surfaced angles a single pass under-weighted (anti-gaming-defenses-worsen-redundancy on karma; over-broad `guilds` scope + unspecified prod-to-API data provenance on oauth).

## What it did NOT settle (the headline limits)
- **Correctness is unmeasured.** I measured that the panel *changes* verdicts (always toward HOLD) and that judges *slightly prefer* its reports. I did **not** measure that the panel is more **correct** — there is no ground truth for these ideas, and "more conservative" could be accuracy OR over-caution. The case that a flipped verdict is worth its cost rests partly on JUDGMENT (a false-BUILD wastes a downstream lane session), labelled as such below.
- **Not idea-engine's actual battery.** This is a faithful *reconstruction* of the mechanism, not the repo's real 8-question prose battery or its (unspecified) real persona roster; ideas were probed cold.
- **Quality margins lean on same-family LLM judges** (Opus judging Opus) → possible elaborateness bias inflating the "slight" edge; the categorical **flip** finding does not depend on the judges and is the robust core.
- **N-sensitivity, persona-roster sensitivity, and mode-3 (forward-sim) are untested.** N was fixed at 3.

## The validity gate (all five, quoted verbatim, answered honestly)
**1. COMPARABLE TO LIVE? "what the model abstracts away, and whether any gap could flip the conclusion"** — Abstracts: (a) a reconstructed battery/roster, not idea-engine's real prompts; (b) Opus-4.8 in this environment, so absolute token/wall costs differ from a live deployment though the ratios should largely transfer; (c) one idea (karma) was thinner than the other two. The measured *direction* — panel flips contested ideas toward conservatism at ~3-4× cost — is a structural property of adding skeptical lenses + synthesis and is unlikely to reverse under the real prompts; the exact flip rate and which ideas count as "contested" would move. **Partially comparable** — a mechanism measurement, not a substitute for running idea-engine's real battery on its real backlog at its deployment model.
**2. UNCORRUPTED? "no bugs (self-check the sim), no seeded luck (multiple seeds / statistical stability), no parameter cherry-picking (report the sweep, not the best point)"** — Instrumentation is real: every agent exposed a `<usage>` block, no numbers were invented, and `analyze.py` asserts panel cost > single-pass and validates every enum/range (exit 1 on any violation). Seeded luck is handled by 3 reps/mode/idea plus the explicit **intra-vs-inter-mode control** (the karma flip survives it). No cherry-picking: **all 18 cells and all 6 judge records are reported**, including the one no-flip idea. Residual risk: the judges share the probes' model family (elaborateness bias) — so the "slight" quality margins are discounted and the categorical flip is treated as load-bearing.
**3. ROBUST? "does the conclusion survive variation at the edges"** — Across 3 reps each, the pattern held: flips clustered on the contested ideas, cost ratios were stable across all three ideas (tokens 3.0-3.1×, wall ~1.6×), and both flips ran the same direction. Not varied: N (fixed 3), persona roster, model, battery wording, idea count (3). **Robust within the tested envelope**; sensitivity to N / roster is unmeasured.
**4. REPRODUCIBLE? "committed code, one documented command, same result"** — The live probe layer is **not** bit-reproducible (LLM nondeterminism) — stated plainly, not hidden. What is reproducible: the 18 frozen cells + 6 judge records are committed under `runs/`, and `python3 sims/intake-001-probe-panel-vs-single-pass/analyze.py` deterministically recomputes every headline number, runs the self-checks, and exits 0 — identical every run. The exact prompts are committed in `probe_protocol.md` so anyone can *re-measure* (a similar distribution, not identical numbers).
**5. LIMITS? "what this evidence does NOT show"** — Verdict *correctness* is unmeasured (only verdict-change + slight quality preference are); this is not idea-engine's real battery; same-family judge bias may inflate the quality edge; N / roster / mode-3 sensitivity is untested; whether the panel's extra conservatism is ever *wrong* (false HOLDs / over-caution) is not adjudicable without ground truth. Absolute costs are environment-specific; only the ratios transfer.

## EVIDENCE STRENGTH: **moderate**
The categorical findings — panel flips 2/3 contested ideas toward conservatism; the karma flip beats intra-mode noise; cost ratios 4× agents / 3.05× tokens / 1.61× wall; judges 6/6 with no position bias — are genuinely measured on real ideas with real instrumentation and survive 3 reps. Strength is held to *moderate*, not *strong*, by n=3 ideas × 3 reps, a single model, a reconstructed (not live) battery, and — the sharpest cut — that verdict **correctness** is unmeasured. **Gate: PASSES** for the claims it covers (verdict-change + cost + quality-preference); the correctness question is out of scope and named as such, not a failed claim.

## VERDICT & recommendation (for the fleet manager to route)
- **Verdict: `approve` — selectively.** Adopt panel-mode **only for big-or-contested ideas** (the README's current default, now measured); **reject always-on**. The "reject always-on" half is directly evidenced: on the well-scoped idea the panel spent ~3× the tokens to reach the *same* BUILD verdict with only a slight, possibly-biased quality gain.
- **Target repo:** `menno420/idea-engine` (owns the battery + probe flow per the proposal's Q7). superbot builds nothing new.
- **Recommended implementation:** keep mode 1 as the default probe; **escalate to panel when EITHER** (a) a quick 2-3× repeat of the single pass **disagrees with itself** (intra-mode instability = ambiguity — empirically the `discord-oauth` signature), **OR** (b) the idea carries **irreversible / high-blast-radius surface** (security, data, spend, public exposure). Route single-pass-stable, low-stakes ideas straight through mode 1. Fix N=3 with distinct lenses (skeptic / builder / user-value) + a synthesizer; log agents/tokens/wall per probe so the escalation policy can be tuned against evidence.
- **Guardrails to ship with it:** (1) the honesty norm ("not measured" beats invention) must survive synthesis — the synthesizer inherited it here; (2) because the panel *always* moved toward HOLD, track whether panel-HOLD ideas that shipped anyway later succeeded, to catch over-caution.
- **Telemetry the build MUST log:** per probe — mode, agents, tokens, wall; per escalation — which trigger fired; per verdict — whether the panel changed the single-pass recommendation. That closes the one gap this prototype could not: verdict **correctness** over time.
