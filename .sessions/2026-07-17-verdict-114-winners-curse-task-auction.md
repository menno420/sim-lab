## 💡 Session idea
The cure here — shade by w·(N−1)/(N+1) — is SEED-pinned to KNOWN N and KNOWN noise-scale w, but the operational fleet failure mode is that an agent rarely knows how many RIVALS it faces: entry is endogenous and the realized bidder count N is a random variable the agent must forecast. Because the optimal shade is convex-increasing in N and the residual curse is asymmetric — UNDER-estimating N under-shades and leaves you exposed to the full order-statistic bias of the true (larger) field, while OVER-estimating N merely forgoes bounded profit — an agent that anchors on an optimistic "few rivals" prior systematically overpays. The natural next FLEET-external slice pins an N-uncertainty model: draw the true field size N ~ (e.g.) Poisson/truncated, let the agent shade for its EXPECTED field size Ê rather than realized N, and pre-register TWO gates: (1) an agent that shades for a downward-biased Ê < E[N] earns mean winner profit strictly negative at ≥3σ (the curse survives shading when N is mis-forecast low), and (2) the residual loss WIDENS monotonically as the forecast bias E[N]−Ê grows, verified across a bias ladder — converting the one-number "shade by w·(N−1)/(N+1)" rule into the actionable "shade for the field you'll actually face, and when unsure round N UP, not down." That is a genuinely different mechanism (competitor-count forecast risk) from this slice's known-N order-statistic correction, seed-pinnable and ≥3σ-gateable.
