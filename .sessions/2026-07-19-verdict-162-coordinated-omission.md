Reproduce PROPOSAL 149 (round-35 FLEET slot, P149 ‚Üí V162, +13): coordinated omission. One single-server FIFO system is measured two ways over the SAME stall schedule and the SAME per-request service times. The folk belief: "my load generator drives the system at rate Œª and records every request's latency, so the p99 it prints IS the tail my users feel; a closed-loop harness (fire a request, wait for the response, fire the next) is a faithful sampler." It is NOT: a closed loop can only ever have ONE request outstanding, so when the system stalls for a duration D it fires one request that blocks for the whole stall and issues NOTHING else ‚Äî it records ~1 inflated sample and OMITS the ~D/œÑ requests an open-loop drive at rate Œª would have piled behind the stall. Those omitted samples are precisely the ones that measure the tail, and they are missing because the generator's pacing is coordinated with (backs off during) the very stalls it should measure ‚Äî coordinated omission. The reported tail is therefore biased LOW by orders of magnitude: the closed p99 is set by the fast between-stall path (~sub-ms) while the true open-loop p99 sits deep in the stall tail (~D).
