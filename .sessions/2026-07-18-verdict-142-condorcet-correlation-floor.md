# VERDICT 142 — condorcet jury correlation floor

Reproduce PROPOSAL 129 (round-30 FLEET slot, P129 → V142, +13): the correlated-vote breakdown of the Condorcet Jury Theorem — multi-agent binary-decision aggregation / social-choice reliability. N agents each independently correct with marginal probability p>1/2 decide a binary question by MAJORITY VOTE. The Condorcet Jury Theorem (Condorcet 1785) promises the majority is correct with probability → 1 as N → ∞ — but only UNDER INDEPENDENCE. Real fleets share an error source (one stale upstream, one shared prior, one correlated feature). Model that with a one-factor latent-Gaussian probit: agent i's latent score X_i = √ρ·C + √(1−ρ)·ε_i with C, ε_i ~ N(0,1) iid, vote CORRECT iff X_i > τ, τ = Φ⁻¹(1−p) so the marginal P(correct) = Φ(−τ) = p for every agent regardless of ρ, and the pairwise latent error-correlation is ρ. Condition on the common factor C: votes are independent with per-agent correct-prob q(C) = Φ((√ρ·C − τ)/√(1−ρ)); by the LLN the fraction correct → q(C), so as N → ∞ the majority is correct iff q(C) > 1/2 iff C > τ/√ρ — the whole jury collapses onto which side of a threshold the shared factor C landed. Hence the majority accuracy has a CLOSED-FORM CEILING A_∞ = P(C > τ/√ρ) = Φ(Φ⁻¹(p)/√ρ) < 1 for every ρ>0, with A_∞ → 1 only as ρ → 0 (the Condorcet limit). Correlation does not merely SLOW the climb, it CAPS it: accuracy converges to a value strictly below certainty, set by p and ρ, and no number of voters can push past it — the effective crowd saturates near N ≈ 1/ρ. On the pinned world (p=0.60, ρ=0.30 ⇒ A_∞ = Φ(Φ⁻¹(0.6)/√0.3) = 0.678155) a 501-agent correlated jury reaches only ~0.6786 while the SAME agents made independent reach ~0.99998 — 500 correlated 60%-experts are only ~68% accurate, the residual ~32% error unremovable by more voters. The operator lesson: a majority vote is only as good as the INDEPENDENCE of its voters; measure the residual error-correlation ρ (not just the count), read Φ(Φ⁻¹(p)/√ρ) as your reliability ceiling, and to raise it DECORRELATE the voters (independent inputs/priors/data paths) — adding more correlated voters cannot.
