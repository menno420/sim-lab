## What happened

OWNER-DIRECT intake (not the idea-engine queue): the owner asked for an audit of the fleet's four
websites (`menno420/websites` @ `31cfd9f`) — do they serve their stated purpose, is the navigation
healthy (dead links / orphans / dead-ends / broken assets), and are they consistent with each other.

Four crawl snapshots already existed under `runs/`: three LIVE crawls of the production Railway URLs
(control-plane, botsite, dashboard) + one LOCAL build of the not-yet-deployed `review` service.
This session wrote `analyze_nav.py` (stdlib-only, vendor-copied harness helpers), which loads all
four snapshots and RE-DERIVES total pages, depth, dead links, broken assets, orphan pages
(inbound-graph), dead-end pages, console errors, forms, task pass rate, re-crawl agreement, and a
cross-site consistency block — running the whole computation twice and asserting byte-identical
results. Also wrote `crawl_site.py`, a clean stdlib BFS reference crawler documenting the re-crawl
command (live crawl is agreement-reproducible, not seed-deterministic).

**Headline measured results (243 self-checks, 0 failed):** all four sites serve their purpose with
REAL (non-placeholder) data and healthy navigation. The one substantive defect: **control-plane has
25 dead in-content links (all 404)** — markdown `.md`/`.sh` paths rendered as site-relative URLs on
the /environments,/queue,/reviews,/fleet detail pages. The one structural gap: **review is NOT
deployed** ("Built for Anthropic reviewers" but reviewers have no URL — owner-action). botsite (0
defects, 480 assets clean) and dashboard (0 forms — read-only proven, /admin honestly stubbed) are
clean; both large sites were truncated by the 80-page cap (disclosed limit). Cross-site: the `ds/`
design system is shared by 3 of 4 — control-plane is the styling outlier.

**Playwright wall (recorded as a LIMIT):** on all 3 LIVE sites Playwright was SKIPPED — Chromium
build mismatch (pw 1.61 wants build 1228, only 1194 present; `playwright install` disallowed) AND
every `page.goto` returned `net::ERR_CONNECTION_RESET` through the agent proxy. Real-browser console
errors were captured ONLY for review locally (6, all benign). The stdlib HTTP passes are
authoritative for the nav graph (sites are no-JS server-rendered).

**Verdict:** approve (serves-purpose on all four) + ship the named fixes; evidence label MEASURED
(numeric crawl simulation for 3 live sites + measured prototype for the local review build); gate PASS.
